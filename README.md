# PPOkemon

Pokemon Showdown 배틀 환경을 기반으로 PPO(Proximal Policy Optimization) 알고리즘을 통해 배틀 전략을 학습하는 강화학습 프로젝트입니다.
Stable-Baselines3의 MaskablePPO를 사용하며, 직접 작성하는 코드는 모두 이 폴더에 위치합니다.

---

## 제한 환경

전체 포켓몬 풀을 사용하면 학습 시간과 자원이 부족하므로 다음과 같이 제한합니다.

| 항목 | 제한 |
|------|------|
| 포켓몬 수 | 9마리 (단일 타입만) |
| 배틀 형식 | 3 vs 3 |
| 기술 | 공격기만 허용 (변화기 제외), 타입 견제기 필수 포함 |
| 아이템 | 사용 안 함 |
| 특성 | 배틀에 영향을 주지 않는 특성만 사용 |
| 상태이상 | 고려하지 않음 (랭크 변화 사이드이펙트는 존재) |

**학습 목표**: 에이전트가 타입 상성과 교체 타이밍을 스스로 학습

---

## State (관측 공간) 정의

### 개요

`envs/observation.py`에 정의된 **70차원 float32 벡터**를 사용합니다. 값의 범위는 `[-1.0, 1.0]`입니다.

포켓몬의 타입 정보는 state에 직접 인코딩하지 않습니다. 대신 9마리 제한 풀 내에서의 **포켓몬 ID를 one-hot**으로 인코딩하여, 신경망이 ID와 타입 상성의 관계를 경험을 통해 학습하도록 합니다.

### 벡터 레이아웃

```
[0  : 41]  내 팀
[41 : 82]  상대 팀
```

각 팀은 동일한 구조로 이루어집니다.

```
팀 (35차원)
├── 액티브 슬롯  (15차원)  [0:15]
└── 벤치 슬롯 2개 (10차원 × 2)  [15:35]
```

### 액티브 슬롯 (15차원)

| 요소 | 차원 | 범위 | 설명 |
|------|------|------|------|
| HP 비율 | 1 | [0, 1] | `current_hp / max_hp` |
| 포켓몬 ID | 9 | {0, 1} | 9마리 풀 내 인덱스 one-hot |
| 스탯 변화량 | 5 | [-1, 1] | atk / def / spa / spd / spe 각각 ÷ 6 정규화 |

기술 위력은 포켓몬 풀 정의 파일에서 관리하므로 state에 포함하지 않습니다.

### 벤치 슬롯 (10차원)

| 요소 | 차원 | 범위 | 설명 |
|------|------|------|------|
| HP 비율 | 1 | [0, 1] | 기절 시 0 → 슬롯 전체가 0 벡터 |
| 포켓몬 ID | 9 | {0, 1} | one-hot |

`hp_ratio = 0`이 기절을 의미하므로 별도의 `is_alive` 플래그는 중복입니다.

### 전체 구조 요약

```
인덱스       내용
──────────────────────────────────────────────────
[0  :  1]   내 액티브 HP
[1  : 10]   내 액티브 포켓몬 ID (one-hot 9)
[10 : 15]   내 액티브 스탯 변화 (atk/def/spa/spd/spe)
[15 : 25]   내 벤치 #1 (hp + id)
[25 : 35]   내 벤치 #2 (hp + id)
[35 : 50]   상대 액티브 슬롯 (같은 구조)
[50 : 60]   상대 벤치 #1
[60 : 70]   상대 벤치 #2
```

### 설계 결정 사항

- **타입·기술 위력 제외**: 타입과 기술 정보는 `pokemon_pool/pokemon_list.py`에서 관리합니다. 에이전트는 포켓몬 ID 간의 상성과 기술 효율을 보상 신호로 학습합니다.
- **PP 비율 제외**: PP 관리는 학습 목표와 무관하므로 state에서 제외하고 액션 마스킹으로만 처리합니다.
- **스탯 변화 포함**: 기술의 부가 효과로 인한 랭크 변화(공격력 상승 등)가 존재하므로 포함합니다.
- **벤치에 스탯 변화 미포함**: 교체 시 스탯 변화가 초기화되므로 벤치 슬롯에는 불필요합니다.

## Action (행동 공간) 정의

### 개요

`envs/spaces.py`에 정의된 **`Discrete(6)`** 이산 행동 공간을 사용합니다.

3v3 배틀에서 한 턴에 선택할 수 있는 행동은 **기술 사용** 또는 **포켓몬 교체** 둘 중 하나입니다.

```
기술 4개  +  교체 대상 2마리  =  총 6개  →  Discrete(6)
```

> State와 Action은 모두 `envs/spaces.py`에서 함께 관리합니다.

### 행동 인덱스 레이아웃

| 인덱스 | 슬롯 | 설명 |
|--------|------|------|
| 0 | move slot 0 | active 포켓몬의 첫 번째 기술 사용 |
| 1 | move slot 1 | active 포켓몬의 두 번째 기술 사용 |
| 2 | move slot 2 | active 포켓몬의 세 번째 기술 사용 |
| 3 | move slot 3 | active 포켓몬의 네 번째 기술 사용 |
| 4 | switch slot 0 | 벤치 포켓몬 #0으로 교체 |
| 5 | switch slot 1 | 벤치 포켓몬 #1으로 교체 |

### Action Masking

매 턴 실제로 사용 가능한 action은 6개보다 적을 수 있으므로, PPO에게 불법 행동을 차단하는 boolean 마스크를 함께 제공합니다.

| 상황 | 마스킹 결과 |
|------|-------------|
| 기술이 3개뿐인 포켓몬 | `mask[3] = False` |
| 벤치 포켓몬 1마리 기절 | `mask[5] = False` |
| 강제 교체 턴 (active 기절 직후) | `mask[0~3]` 전부 `False` |

`mask=False`인 슬롯에는 PPO의 softmax 직전에 `-inf`를 더해 선택 확률을 0으로 만듭니다.

**강제 교체 턴**: active 포켓몬이 기절했을 때 발생합니다. 이 턴에는 기술 사용이 불가능하며 교체만 허용됩니다. `battle.force_switch` 플래그로 구분합니다.

### 주요 함수

| 함수 | 역할 |
|------|------|
| `build_action_mask(battle)` | 현재 턴의 유효 행동 마스크 `(6,) bool` 반환 |
| `action_to_order(action, battle)` | 정수 action → poke-env `BattleOrder` 변환 |
| `safe_action(action, battle)` | 마스크 위반 시 유효 행동으로 폴백 (디버깅 전용) |

### 설계 결정 사항

- **`Discrete(6)` 선택**: `MultiBinary`나 `Dict` 공간 대신 단순 정수 공간을 사용하여 `MaskablePPO`와의 호환성을 유지합니다.
- **기술 슬롯 고정 (4개)**: 실제 기술 수가 4개 미만인 경우 masking으로 처리하며, 공간 크기는 항상 6으로 고정합니다.
- **교체 슬롯 2개**: 3v3에서 벤치 크기는 2이므로 switch slot은 2개입니다.
- **`force_switch` 분리 처리**: 강제 교체 턴과 일반 턴을 명시적으로 구분하여 불법 action 선택 가능성을 차단합니다.
